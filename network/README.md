# Network examples

This directory contains public, placeholder-only examples for host-level network
configuration. Live tunnel IDs, credential paths, ingress domains, addresses, and
host inventory are operator-local data and must not be committed.

## cloudflared

Copy [`cloudflared/config.example.yml`](cloudflared/config.example.yml) to the
ignored `cloudflared/config.yml`, then replace the placeholder UUID, hostname, and
service URL. The credentials JSON is a secret generated by Cloudflare and belongs
only on the target host.

```bash
cp network/cloudflared/config.example.yml network/cloudflared/config.yml
$EDITOR network/cloudflared/config.yml
cloudflared tunnel ingress validate
```

Applying a tunnel change is intentionally manual because it changes externally
reachable infrastructure. One generic sequence is:

```bash
scp network/cloudflared/config.yml brokkr@control-node:/tmp/cloudflared.yml
ssh brokkr@control-node \
  'sudo install -m 600 /tmp/cloudflared.yml /etc/cloudflared/config.yml && \
   sudo systemctl restart cloudflared'
ssh brokkr@control-node 'systemctl status cloudflared --no-pager'
```

Validate that the local config contains no live values before committing. The live
`config.yml` is ignored, so only updates to `config.example.yml` should appear in Git.
